# Configuration

This CLI keeps its configuration inside a local `.env` file so you can switch accounts without changing source control.

## Files

- `.env.example`: copy this to `.env` and fill it before running the CLI (do not commit `.env`).
- `.state/token.json`: optional storage for OAuth tokens generated by the CLI (always gitignored).

By default the CLI reads values from `.env`. Any values you export into the OS environment will override the file values, which is helpful for CI or temporary credentials.

## Environment variables

- `OPENAI_API_BASE_URL` (required): HTTP base URL for the OpenAI API. The sample file sets this to `https://api.openai.com/v1` so you can target standard OpenAI hosts.
- `OPENAI_API_KEY` (required): your OpenAI API key or organization-wide service key. The CLI only sends this via an `Authorization: Bearer` header.
- `OPENAI_ORGANIZATION_ID` (optional): include this header when your account requires organization scoping.
- `OPENAI_PROJECT_ID` (optional): include this header to scope calls to a specific project.
- `OPENAI_TIMEOUT_S` (optional, defaults to `30`): HTTP request timeout in seconds.

The CLI redacts any API key or token before logging, plans, refusals, or receipts so secrets never appear in audit artifacts.
