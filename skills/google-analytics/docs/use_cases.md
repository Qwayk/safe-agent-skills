# Use cases

Google Analytics is useful when you want to understand what people did on your site, where they came from, and whether the important actions are being measured correctly.

Ask the agent for answers you would normally hunt for across GA4 screens: which pages are getting attention, which traffic sources are bringing useful visitors, whether conversions are firing, who has access, and whether the property setup looks clean.

## Good questions to ask

- "What changed on the site in the last 7 days?"
- "Which pages brought the most engaged visitors, not just the most visits?"
- "Which traffic sources are sending people who actually convert?"
- "Are our key events, conversions, audiences, and custom dimensions set up correctly?"
- "Who has access to this account or property, and does anything look risky?"
- "Did someone change the property, stream, audience, or conversion setup recently?"
- "Can you compare this week to last week and tell me what matters?"
- "Can you check the data stream and linked products before we touch anything?"

## Everyday work this helps with

### Weekly site review

The agent can pull a clean report for traffic, engagement, conversions, sources, devices, countries, and top pages. This is useful before a client call, launch review, or Monday planning check.

### Conversion and tracking checks

Ask the agent to review conversion events, audiences, custom dimensions, custom metrics, and data streams. It should explain what looks complete, what looks missing, and what needs a person to confirm.

### Access and account cleanup

The agent can list accounts, properties, access bindings, and access reports so you can see who can view or manage Analytics data.

### Change review

If you want an admin change, ask the agent to show the current target first, then prepare the plan. You should be able to see the account, property, stream, audience, event, or access target before anything changes.

## What the agent should show you

- The account and property it is using.
- The date range, metrics, and dimensions in any report.
- The plain-English answer before the raw JSON.
- The setup item it checked, such as a stream, conversion event, audience, custom dimension, or linked product.
- A clear note when missing access or unclear property IDs block the work.
- A review plan before any GA4 admin change.

## Good first path

Start with: "Check the Google Analytics connection, list the accounts and properties I can access, then show me a simple last-7-days report for traffic, engagement, and conversions."

After that, ask one practical follow-up, such as: "Which source brought the best converting visitors?" or "Are the conversion events set up well enough to trust this report?"
