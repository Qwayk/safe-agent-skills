# API coverage (endpoints → CLI)
Purpose:
- Make the Cloudinary REST coverage claim exact and auditable.
- Keep one active main reference for the runtime allowlist.
- Show which official REST groups are shipped, gated, beta, or out of scope.

## Summary

- Provider: `Cloudinary`
- API base URLs:
  - Product v1: `https://<product-host>/v1_1/<cloud_name>`
  - Product video v2: `https://<product-host>/v2/video/<cloud_name>`
  - Product analysis v2: `https://<product-host>/v2/analysis/<cloud_name>`
  - Account provisioning: `https://<account-host>/v1_1/provisioning/accounts/<account_id>`
  - Account permissions: `https://<account-host>`
- Auth methods:
  - Product APIs: HTTP Basic Auth with `CLOUDINARY_API_KEY` + `CLOUDINARY_API_SECRET`
  - Unsigned upload: no secret; upload preset still required by Cloudinary
  - Account APIs: HTTP Basic Auth with `CLOUDINARY_ACCOUNT_API_KEY` + `CLOUDINARY_ACCOUNT_API_SECRET`
  - Public Permissions utilities: no auth required
- Last audited (UTC): `2026-05-25`
- Official REST operations shipped: `175`
- Non-REST official reference excluded: `Transformation URL API`
- Current write safety: each non-read operation can create a plan, but write apply attempts require explicit no-snapshot approval before Cloudinary HTTP until command-specific saved snapshot support is available. The safety column below lists the gates needed before that refusal.
- There is no generic rollback or snapshot command exposed by this tool. Backup download is read-only; restore and backup-delete writes currently require explicit no-snapshot approval before Cloudinary HTTP when no saved snapshot is available.

## Known gaps (explicit)

- `Transformation URL API` — excluded by product choice because this tool is for Cloudinary's official REST APIs, not URL-generation syntax.
- Live verification is still needed with real Cloudinary credentials for product, account, and gated account features. The shipped runtime and tests are local and doc-driven.
- Provisioning and non-public Permissions endpoints are access-gated by Cloudinary account plan or enablement. They are shipped and documented, but may be blocked live without the right Cloudinary account access.
- No global rollback/snapshot API is listed as part of shipped coverage. Backup download is read-only; write restore/delete pathways are listed above but currently require explicit no-snapshot approval before Cloudinary HTTP.
