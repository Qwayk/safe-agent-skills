# What you can do with Amazon Product Advertising API

Amazon Product Advertising API is useful when you need real Amazon catalog data before you write, compare, recommend, or build affiliate links.

Ask the agent for the work an affiliate writer, editor, or product researcher does before publishing: find product candidates, confirm exact ASINs, compare details, clean up Amazon URLs, check categories, and build links only after the product match looks right.

## Good questions to ask

- "Which products are worth considering for this gift guide?"
- "Can you search Amazon for five strong options in this niche?"
- "Can you fetch details for these ASINs and summarize the useful differences?"
- "Can you turn these messy Amazon URLs into clean ASINs?"
- "Can you build affiliate links for these confirmed products?"
- "Which browse node or category should I use for this product research?"
- "Can you run this product research CSV and show which rows failed?"
- "Do these products look like real matches for the article, or are they too broad?"

## Everyday work this helps with

### Product shortlist research

The agent can search Amazon by keyword, pull structured product details, and help you narrow a messy result set into a practical shortlist for an article, review, comparison table, or buying guide.

### ASIN and URL cleanup

Ask the agent to resolve Amazon product links into clean ASINs before you reuse them. This is useful when source notes, spreadsheets, or old drafts contain copied product URLs.

### Affiliate link preparation

The agent can build affiliate links from products you already trust. It should not treat the link as the decision; it should first confirm the ASIN, marketplace, and partner tag assumptions.

### Repeatable research batches

If you have a CSV of products, keywords, or ASINs, the agent can run a batch and tell you what worked, what failed, and which rows need a human look.

## What the agent should show you

- The marketplace, partner tag, keyword, ASIN, URL, or browse node it used.
- A short reason each product looks useful for the job.
- Product details that matter for comparison, not every raw field.
- Failed rows or missing products in plain English.
- Clean affiliate links only after the product match is confirmed.
- A small first pull before a larger batch when the scope is uncertain.

## Good first path

Start with: "Check the Amazon Product Advertising skill is configured, search for five products in my niche, show why each one might be useful, and build affiliate links only for the best matches."

After that, ask one practical follow-up, such as: "Fetch details for these ASINs and turn them into a comparison table" or "Resolve these Amazon URLs before I add them to the article."
