# Use cases

This tool helps a publisher work with Awin without needing a custom script each time.

- Check whether your token can see the right publisher account before you trust any later report.
- Find which advertiser programs are joined, pending, rejected, or hidden.
- Pull offers, transactions, and transaction queries into a repeatable workflow instead of clicking around the Awin UI.
- Build advertiser, campaign, and creative performance reads with the same filters every time.
- Generate one tracking link or a prepared batch of links with clear inputs.
- Download enhanced retail feeds and legacy feed exports to local files you can audit later.
- Prepare a proof-of-purchase order batch in dry-run mode, review the saved plan, and only then send it live with the same reviewed plan file.

Why this beats a typical no-code setup:

- It keeps the command list explicit, so each task is predictable.
- It returns structured JSON, which is easier for an AI agent or workflow to verify.
- It keeps proof files and examples local.
- The one write command is review-first instead of sending live data by default, and live apply must reuse the saved plan file.
