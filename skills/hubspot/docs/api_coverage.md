# API coverage (endpoints -> CLI)

Purpose:
- Make the HubSpot CRM surface measurable.
- Lock the shipped command families.
- Keep mixed API versions and gated object types explicit.

Rules:
- This file is the source of truth for the shipped HubSpot command surface.
- Every official CRM endpoint group in scope must map to an explicit CLI action.
- No raw-request bridge counts as coverage.
- If HubSpot keeps an operation on an older official path, keep that path here and note it clearly.

## Summary

- Provider: HubSpot CRM API
- API base URL: `https://api.hubapi.com`
- Default auth method: private app access token in `Authorization: Bearer <token>`
- Optional auth method: manually stored OAuth access token JSON for users who already have one
- Latest stable version used first: `2026-03`
- Older official path still needed: object GDPR delete is still documented on `2025-09`
- Last audited (UTC): `2026-06-04`
- Product scope for this tool:
  - CRM object records and search
  - record associations, labels, and limits
  - object properties and property groups
  - owners
  - pipelines and pipeline stages
  - custom object schemas
  - CRM imports and exports
  - object-library enablement checks

All CLI rows below expand to:

`qwayk-hubspot-safe-agent-cli <CLI command>`

## Scope notes

- `objects` applies to standard objects, custom objects, and activation-gated object types when HubSpot supports them for the account.
- Some HubSpot object types are account-gated or activation-gated. Those rows still count as covered when the CLI exposes the official endpoint and the docs mark live limits honestly.
- `owners` is read-only by provider design.
- `schemas` applies to custom object schemas, not standard object records.
- `imports create` is multipart form-data and needs local files.
- `exports status` can return a short-lived download URL. HubSpot says completed export URLs expire after five minutes.

## Test note

- `tests/test_command_families.py` parses every shipped HubSpot action with the required args.
- `tests/test_api_coverage_alignment.py` checks that `docs/api_coverage.md` has exact one-row-per-action coverage, with no missing or extra CLI rows.
- `tests/test_hubspot_safety.py` covers representative dry-run, `--yes`, `--ack-irreversible`, and apply behavior.
- write apply behavior requires explicit no-snapshot approval before HubSpot HTTP when command-specific before-state capture is not available.
- `tests/test_hubspot_import_validation.py` covers import request-file and upload-file validation.
- The table below is the command-surface proof ledger and is checked directly against shipped actions in an automated exact-match test.
