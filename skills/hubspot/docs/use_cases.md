# Use cases

Use this page when you want ideas for real HubSpot jobs to hand to your agent.
If you need setup first, start with [Connect your account](onboarding.md). If you need exact commands, use [Quickstart](quickstart.md) and [Command reference](command_reference.md).

## Why this helps with real CRM work

Most automation tools are good at one small trigger. This tool is better when you need safe bulk CRM work with a clear preview first.

What makes it useful:
- it can search and review large sets of HubSpot records
- it can show a dry-run before writes
- it can work across records, associations, properties, pipelines, and imports
- it can leave proof files so you can check the plan or safe refusal later

## Common things to ask for

- “Find all companies that match these rules and export the list.”
- “Check which HubSpot objects are active in this account before we use them.”
- “Show me all associations between these contacts and companies.”
- “Create or update these custom properties, but show the plan first.”
- “Review this pipeline, then propose the stage changes as a plan.”
- “Import these records from files and give me the import status and errors.”
- “Create a HubSpot export job and tell me when the download is ready.”
- “Define a new custom object schema and show me the plan without applying it.”

## Why this beats typical no-code work

- Bulk work: change many records or settings in one controlled run.
- Preview first: dry-run plan before any live write.
- Less guessing: the tool uses named HubSpot operations instead of a vague generic call.
- Better proof: runs can save plans, safe refusals, and audit logs.

## What the agent should show you

When a change is requested, the agent should:

1. Show a dry-run preview first.
2. Explain the risk in plain English.
3. Refuse live apply when no saved snapshot is available.
4. Verify with read commands when HubSpot makes that possible.
5. Save or point to the proof files.
