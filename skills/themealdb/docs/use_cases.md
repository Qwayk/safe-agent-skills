# Use cases

Use this page when you want ideas for real TheMealDB jobs to hand to your agent.
If you need setup first, start with [Start with the free public key](onboarding.md). If you need exact commands, use [Quickstart](quickstart.md) and [Command reference](command_reference.md).

## Plain-English outcomes

- See all meal categories
- Find meals by name
- Look up one meal by ID
- Browse available areas and ingredients
- Filter meals by one category, one area, or one ingredient
- Get one random meal idea

## Example requests

- “Show me dessert meals.”
- “Find meals with chicken breast.”
- “List the available meal areas.”
- “Look up meal 52772.”
- “Give me one random dinner idea.”

## Why this beats a typical no-code step

- Clear named commands instead of one hidden raw API call
- Read-only scope, so there is no risk of accidental writes
- Deterministic JSON output for agents and automations
- Proof files and example outputs are easy to audit later
