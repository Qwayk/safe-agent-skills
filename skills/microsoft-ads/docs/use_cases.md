# What you can do with Microsoft Ads

Microsoft Ads work usually starts with a reporting or spend-control question: which account is reachable, what changed in campaigns, which keywords or audiences need review, and what bulk or service-operation change should be previewed before it reaches Bing and Microsoft's ad network.
If you need setup first, start with [Connect your Microsoft Ads account](onboarding.md). If you need exact commands, use [Quickstart](quickstart.md) and [Command reference](command_reference.md).

This skill helps an agent confirm live access, read campaign and report data, research keywords or audiences, and prepare service-operation or bulk plans before anything changes real ad delivery.

## Good jobs to give the agent

### Account and campaign review

- "Check my Microsoft Ads setup and confirm which live reads are safe to run first."
- "Show the account, customer, and campaign targets before we plan any work."
- "Pull a performance report for the last 30 days and save it for review."
- "Review campaigns, budgets, bids, keywords, recommendations, and account settings before any change."
- "Tell me when Microsoft processing time or missing account IDs blocks the next step."

### Keyword, audience, and report work

- "Estimate keyword traffic before we build a plan."
- "Review audience size or targeting data before we change a campaign."
- "Pull a reporting result and explain the useful pattern in plain English."
- "Compare report output with the requested account and date range so we know the target is right."

### Bulk and service-operation planning

- "Prepare a bulk upload or change from this request file and show me the plan first."
- "Preview a campaign-management change and explain the risk before anything goes live."
- "Show the request body for this service operation and tell me whether it is budget, bid, delete-like, or batch work."
- "Explain the approval path for a high-risk budget, bid, or delete-like action."

## What the agent should show you

- Whether the request needs `--live`, and which customer, account, campaign, report, keyword, audience, or bulk file it used.
- The report or account evidence behind the recommendation.
- A dry-run plan before service-operation, bulk, budget, bid, audience, campaign, or delete-like changes.
- Any `--live`, reviewed plan, `--yes`, irreversible, no-snapshot, permission, or missing request-file gate.
- A receipt, refusal, run-history entry, or Microsoft processing note after the request.

## Good first Microsoft Ads path

Start by checking the connection, confirming live access, pulling one small account or performance report, and reviewing the account targets before planning any bulk or service-operation change.
