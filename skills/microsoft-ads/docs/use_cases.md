# Use cases

Use this page when you want practical Microsoft Ads jobs to hand to your agent.
If you need setup first, start with [Connect your Microsoft Ads account](onboarding.md). If you need exact commands, use [Quickstart](quickstart.md) and [Command reference](command_reference.md).

Microsoft Ads work is usually about checking the right account, reading reports, and reviewing the exact request before a service or bulk change.

## Good jobs to give the agent

- "Check my Microsoft Ads setup and confirm which live reads are safe to run first."
- "Pull a performance report for the last 30 days and save it for review."
- "Review campaigns, budgets, bids, or recommendations before we change anything."
- "Estimate keyword traffic or audience size before we build a plan."
- "Prepare a bulk or service-operation change from a request file and show me the plan first."
- "Show me the approval path for a higher-risk budget or delete-like action before anything goes live."

## What the agent should show you

When you ask for a change, the agent should:

1. Confirm live access, account IDs, and request targets before planning.
2. Show the report, campaign, keyword, audience, or bulk context it used.
3. Show the dry-run plan before service-operation, bulk, budget, or delete-like changes.
4. Explain `--live`, plan review, irreversible, and no-snapshot approvals in plain English.
5. Say clearly when Microsoft processing time, permissions, or missing request JSON blocks the work.
