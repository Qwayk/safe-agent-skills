# What you can do with Skimlinks

Skimlinks is useful when a publisher wants to understand where affiliate money is coming from and which merchant links are worth using.

Ask the agent for the work an editor, commerce lead, or affiliate manager checks often: active merchants, commission results, page and link performance, Product Key alternatives, payment status, and monetized links for places where normal Skimlinks JavaScript is not enough.

## Good questions to ask

- "Which merchants are active for this country or vertical?"
- "Which pages or links earned commission last month?"
- "Which merchants or domains should we consider for this article?"
- "Can you group commission results by merchant, page, country, device, or payout type?"
- "Which links or pages look worth updating based on recent earnings?"
- "Does Product Key have alternatives for this item?"
- "Which merchants were deactivated or had payment status changes?"
- "Can you build the official Link Wrapper URL for this destination?"

## Everyday work this helps with

### Merchant research

The agent can search merchants by country, name, vertical, advertiser ID, domain, or offer. This helps before adding links to a new buying guide or checking whether a brand is monetizable.

### Commission and page reporting

Ask the agent to pull commission, link, page, aggregated, payment, or deactivated merchant reports for a clear date range. The useful answer should say what earned, where it came from, and what looks worth reviewing next.

### Product Key checks

When Product Key is enabled, the agent can look up product details and alternatives. It should say clearly when Product Key needs separate access or a publisher domain ID.

### Link Wrapper preparation

The agent can build official Link Wrapper URLs as text. It does not need to open the link; it should show you the destination and generated URL so you can decide where to use it.

## What the agent should show you

- The publisher ID, publisher domain ID, country, date range, merchant filter, or report filter it used.
- A clear merchant, page, link, commission, or product summary before raw data.
- Whether Product Key is available for the account.
- Any report scope that may be broader than expected.
- Link Wrapper URLs as generated text, not as clicked links.
- A note when credentials are valid for one Skimlinks area but not another.

## Good first path

Start with: "Check the Skimlinks skill is configured, show active merchants for my market, and pull last month's top commission links."

After that, ask one useful follow-up, such as: "Which pages should we update based on commission results?" or "Find Product Key alternatives for this product."
